# PLANCK Load-Reduction Pattern Audit

Date: 2026-05-02  
Scope: Verify the listed PLANCK / `activeragdoll` source claims and map each pattern to current ROCK / SCISSORS behavior.  
Method: Source-level audit only. No Ghidra was used because the checked claims all refer to local source code.

## Summary

The PLANCK claims are confirmed against `E:\fo4dev\skirymvr_mods\source_codes\activeragdoll`. Current ROCK/SCISSORS implements the basic actor lifecycle gates, actor-add throttling, provider cleanup, SCISSORS world-recovery delay, bounded point-impulse cooldown pruning, dynamic-push suppression for SCISSORS bodies, and ROCK native melee event suppression. The broad PLANCK load reducers are not yet ported as a complete system: global biped matrix pruning, HIGGS-style filter comparison rules, useful-only self-collision, active-actor physics damage bypass, and listener ordering need explicit FO4VR/hknp equivalents before they should be considered implemented.

## Open Port Gaps

- Design an FO4VR hknp equivalent for PLANCK's biped collision matrix pruning and group-aware collision comparison.
- Add explicit SCISSORS-owned self-collision state before enabling broad ragdoll self-collision.
- Keep SCISSORS damage/event gates disabled until a physics damage bypass and owned damage path are verified.
- Treat hknp contact/listener ordering as an RE question, not as a direct Skyrim hkp copy.
