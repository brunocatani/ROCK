# ROCK Debug Body Overlay

The previous collider debug path cloned visible NIF nodes and transformed them beside the hand skeleton. That made the overlay depend on scene-node parenting, mesh scale assumptions, and duplicated transform math, so it could disagree with the actual `hknpBody` that Havok was solving. The replacement draws the live Havok body by reading the current `hknpBody`, `hknpMotion`, and `hknpShape` data, following the proven CollisionVisualizerF4VR approach while restricting the draw list to bodies ROCK explicitly owns or is currently comparing against.

`bDebugShowColliders` now means "draw ROCK-created physics bodies": hand bodies on layer 43 and weapon bodies on layer 44. `bDebugShowTargetColliders` is a separate comparison overlay for the currently selected or held target body, so hand alignment can be checked against the object being touched or grabbed without turning ROCK into a full-world collision visualizer. The old `iDebugColliderShape` NIF stand-in selector is intentionally removed because the debug mesh now comes from the real `hknpShape`.

`bDebugShowHandAxes` adds transform diagnostics to the same overlay instead of reviving the old NIF marker path. It draws the exact frames ROCK is comparing: raw hand/bone transform, computed ROCK collider transform, live Havok hand body transform, and the selected or held target body when target comparison is enabled. Axis colors are fixed as X red, Y green, Z blue; yellow lines show translation from the raw hand origin to the collider/body center. Different axis lengths identify the sources without text labels: raw is shortest, computed collider is medium, live body is longer, and target body is longest.
